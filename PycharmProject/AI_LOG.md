# AI交互日志
## 项目：豆瓣电影TOP250爬虫

【日志 1】
提问：如何使用 Python 爬取豆瓣电影 TOP250？
AI回答：提供 requests + BeautifulSoup 基础爬虫代码，实现分页爬取、电影名、评分、信息提取，并保存为 CSV 文件。
人工修改：添加 headers 请求头伪装浏览器，解决 403 禁止访问问题。


【日志 2】
提问：如何实现分页爬取 10 页，共 250 条数据？
AI回答：使用 for 循环构造 URL，start 参数从 0 开始，每次增加 25。
人工修改：编写循环逻辑，实现自动翻页爬取全部数据。


【日志 3】
提问：如何防止爬虫被网站封禁？
AI回答：添加 time.sleep(1) 延时，控制访问频率。
人工修改：在每页爬取后加入延时 1 秒，降低请求速度，避免触发反爬机制。


【日志 4】
提问：如何从爬取的信息中提取电影类型（genres）？
AI回答：使用字符串分割 split("/")，从 info 字段中拆分出类型部分。
人工修改：增加容错判断，当格式不满足时自动赋值为“未知”，保证程序不崩溃。


【日志 5】
提问：如何将爬取的数据保存为 Excel 能打开的 CSV？
AI回答：使用 pandas 的 DataFrame 导出，编码使用 utf-8-sig。
人工修改：应用到代码中，确保中文不乱码，可正常用 Excel 打开。


项目总结：
AI 提供了爬虫框架、数据解析、文件保存的核心代码；
人工完成了反爬处理、容错优化、字段扩展（类型genres），最终实现完整可用的爬取程序。


【日志 6】
提问：requests 请求豆瓣返回 403 怎么办？
AI回答：添加请求头 headers 伪装浏览器，同时设置延时 time.sleep(1)，避免频繁请求被限制。


【日志 7】
提问：运行时提示 FileNotFoundError: No such file or directory 怎么办？
AI回答：确保 CSV 文件和代码在同一文件夹，或在 VS 中右键选择「在终端中运行 Python 文件」，让 VS 自动切换到正确路径。


【日志 8】
提问：如何使用 Tkinter 将 CSV 数据显示成表格？
AI回答：使用 ttk.Treeview 组件创建表格，通过 pandas 读取 CSV 文件后循环插入数据。

【日志 9】
提问：如何实现电影名称搜索功能？
AI回答：获取用户输入内容，利用 pandas 的 str.contains() 方法进行模糊匹配筛选数据。
